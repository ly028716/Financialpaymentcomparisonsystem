# 财务付款对比系统 - 需求文档

## 1. 项目背景

### 1.1 业务痛点
- 部门付款申请流程不规范，信息传递易出错
- 会计制作的付款申请表与出纳实际付款存在差异
- 账号、户名、金额错误导致资金风险
- 缺乏自动化校验机制，依赖人工核对效率低

### 1.2 解决方案
构建一套财务付款对比系统，实现：
- 付款申请电子化流转
- 会计审核标准化
- 出纳付款记录数字化
- 自动对比校验，差异预警

---

## 2. 用户角色

| 角色 | 职责 | 权限 |
|------|------|------|
| **部门申请人** | 提交付款申请 | 创建申请、查看自己的申请状态 |
| **会计** | 审核申请、生成付款表 | 审核申请、导出付款清单、查看所有申请 |
| **出纳** | 执行付款、记录实际付款信息 | 查看待付款清单、录入付款记录、上传凭证 |
| **财务主管** | 监督全流程、处理异常 | 查看所有数据、处理差异、生成报表 |
| **系统管理员** | 系统配置、用户管理 | 全部权限 |

## 3. 功能需求

### 3.1 付款申请管理（部门申请人）

#### 3.1.1 创建付款申请
**输入：**
- 收款方户名（必填）
- 收款方账号（必填，自动校验格式）
- 收款方开户行（必填）
- 付款金额（必填，支持大写自动转换）
- 付款用途（必填）
- 附件上传（支持PDF、图片、Word）

**校验规则：**
- 账号格式校验（银行卡号Luhn算法）
- 金额范围校验（0.01 - 999,999,999.99）
- 户名不能包含特殊字符
- 附件大小限制（单个文件≤10MB）

**输出：**
- 生成唯一申请单号（格式：FK+年月日+4位流水号）
- 保存草稿或提交审核

#### 3.1.2 查看申请状态
**功能：**
- 查看自己提交的所有申请
- 查看审核进度（待审核/已通过/已拒绝/已付款/已核验）
- 查看审核意见和付款记录

#### 3.1.3 修改/撤销申请
**规则：**
- 仅"草稿"和"已拒绝"状态可修改
- 仅"待审核"状态可撤销
- 已付款的申请不可修改/撤销

---

### 3.2 会计审核管理

#### 3.2.1 审核付款申请
**功能：**
- 查看待审核申请列表
- 审核申请详情（收款方信息、金额、附件）
- 通过/拒绝申请（拒绝需填写原因）

**审核检查项：**
- [ ] 收款方信息完整准确
- [ ] 账号格式正确
- [ ] 金额大小写一致
- [ ] 附件齐全（合同、发票等）
- [ ] 预算额度充足
- [ ] 收款方不在黑名单

#### 3.2.2 生成付款申请表
**功能：**
- 批量选择已审核通过的申请
- 生成标准付款申请表（Excel/PDF）
- 导出格式可配置

**付款申请表模板：**
```
序号 | 申请单号 | 申请部门 | 收款户名 | 收款账号 | 开户行 | 金额 | 用途 | 备注
1    | FK...   | 技术部   | XX公司  | 622...  | 工行   | 50000| 采购 |
```

#### 3.2.3 批量导出
**功能：**
- 按日期范围导出
- 按部门导出
- 按状态导出
- 支持Excel、PDF、CSV格式

---

### 3.3 出纳付款管理

#### 3.3.1 查看待付款清单
**功能：**
- 查看会计审核通过的待付款申请
- 按优先级排序（紧急程度、金额大小）
- 筛选功能（部门、金额范围、日期）

#### 3.3.2 记录实际付款
**输入：**
- 实际收款户名（必填）
- 实际收款账号（必填）
- 实际付款金额（必填）
- 付款渠道（网银/柜台/支票等）
- 银行流水号（选填）
- 付款凭证上传（必填，银行回单截图）

**功能：**
- 支持OCR识别银行回单（自动提取户名、账号、金额）
- 手工录入或OCR识别二选一
- 实时保存，支持分批付款

#### 3.3.3 批量付款
**功能：**
- 批量选择多笔付款
- 批量上传付款凭证
- 批量标记完成

---

### 3.4 智能对比校验（核心功能）

#### 3.4.1 自动对比
**触发时机：**
- 出纳提交付款记录后自动触发
- 会计/财务主管手动触发重新对比

**对比逻辑：**
```
1. 账号对比（完全匹配）
   - 会计申请表账号 == 出纳实际账号
   - 不匹配 → CRITICAL级别预警

2. 户名对比（模糊匹配）
   - 去除空格、括号、"有限公司"等后缀
   - 计算相似度（Levenshtein距离）
   - 相似度 < 95% → HIGH级别预警

3. 金额对比（精确匹配）
   - 误差 ≤ 0.01元视为匹配
   - 不匹配 → CRITICAL级别预警

4. 开户行对比（关键词匹配）
   - 提取银行名称关键词（工行、建行、农行等）
   - 关键词不匹配 → MEDIUM级别预警
```

#### 3.4.2 差异报告
**内容：**
- 对比结果（匹配/不匹配）
- 差异明细表

| 对比项 | 会计申请 | 出纳实际 | 差异 | 严重程度 |
|--------|---------|---------|------|---------|
| 账号 | 622... | 622... | 一致 | - |
| 户名 | 北京XX公司 | 北京XX科技公司 | 不一致 | HIGH |
| 金额 | 50000.00 | 50000.00 | 一致 | - |

**预警通知：**
- 系统内消息（红色标记）
- 邮件通知（会计、出纳、财务主管）
- 钉钉/企业微信机器人推送
- 短信通知（CRITICAL级别）

#### 3.4.3 人工复核
**功能：**
- 财务主管查看差异报告
- 判断差异是否合理（如：户名简称、分批付款等）
- 标记为"正常差异"或"异常差异"
- 填写复核说明

---

### 3.5 报表统计

#### 3.5.1 付款统计报表
**维度：**
- 按部门统计（申请笔数、金额）
- 按时间统计（日/周/月/年）
- 按状态统计（待审核/已付款/有差异）

#### 3.5.2 差异分析报表
**内容：**
- 差异发生率（差异笔数/总笔数）
- 差异类型分布（账号错误/户名错误/金额错误）
- 高频错误收款方
- 出纳错误率排名

#### 3.5.3 效率分析报表
**指标：**
- 平均审核时长
- 平均付款时长
- 流程总耗时
- 各环节瓶颈分析

## 4. 非功能需求

### 4.1 性能要求
- 页面响应时间 < 2秒
- 支持并发用户数 ≥ 100
- 单次批量导出 ≤ 10000条记录
- OCR识别时间 < 5秒

### 4.2 安全要求
- 用户登录采用双因素认证（密码+短信验证码）
- 敏感数据加密存储（账号、金额）
- 操作日志完整记录（谁、何时、做了什么）
- 权限控制（RBAC模型）
- 数据备份（每日全量备份）

### 4.3 可用性要求
- 系统可用性 ≥ 99.5%
- 故障恢复时间 < 1小时
- 数据不丢失

### 4.4 兼容性要求
- 浏览器：Chrome、Edge、Firefox、Safari（最新版本）
- 移动端：支持响应式设计，适配手机/平板
- 操作系统：Windows、macOS、Linux

### 4.5 易用性要求
- 界面简洁直观，符合财务人员使用习惯
- 关键操作有二次确认
- 错误提示清晰明确
- 提供操作手册和视频教程

## 5. 业务规则

### 5.1 申请单号规则
- 格式：FK + YYYYMMDD + 4位流水号
- 示例：FK202604060001
- 每日流水号从0001开始

### 5.2 金额规则
- 最小金额：0.01元
- 最大金额：999,999,999.99元
- 精度：保留2位小数
- 大写金额自动转换

### 5.3 审核规则
- 单笔金额 < 10万：会计审核即可
- 单笔金额 ≥ 10万：需财务主管二次审核
- 单日累计金额 ≥ 100万：需总经理审批

### 5.4 付款规则
- 付款必须在审核通过后3个工作日内完成
- 超期未付款自动提醒
- 紧急付款可标记优先级

### 5.5 对比规则
- 账号必须完全匹配
- 户名相似度 ≥ 95%视为匹配
- 金额误差 ≤ 0.01元视为匹配
- 开户行关键词匹配即可

## 6. 数据字典

### 6.1 申请状态
| 状态码 | 状态名称 | 说明 |
|--------|---------|------|
| draft | 草稿 | 申请人保存但未提交 |
| pending | 待审核 | 已提交等待会计审核 |
| approved | 已通过 | 会计审核通过 |
| rejected | 已拒绝 | 会计审核拒绝 |
| paid | 已付款 | 出纳已完成付款 |
| verified | 已核验 | 系统对比完成 |

### 6.2 差异严重程度
| 级别 | 说明 | 处理方式 |
|------|------|---------|
| CRITICAL | 严重（账号、金额错误） | 立即通知，禁止继续流程 |
| HIGH | 高（户名不匹配） | 通知复核 |
| MEDIUM | 中（开户行不匹配） | 记录日志 |
| LOW | 低（付款延迟） | 仅提醒 |

### 6.3 付款渠道
- 网银转账
- 柜台转账
- 支票
- 承兑汇票
- 其他

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| OCR识别准确率低 | 中 | 提供手工录入兜底方案 |

| 数据安全风险 | 高 | 加密存储，权限控制，审计日志 |
| 系统性能瓶颈 | 中 | 分批处理，异步任务，缓存优化 |
