## 1. 接口设计概述

### 1.1 RESTful API设计原则

遵循RESTful规范，使用标准HTTP方法：

| HTTP方法 | 操作 | 示例 |
|---------|------|------|
| GET | 查询资源 | `GET /api/applications` 获取申请列表 |
| POST | 创建资源 | `POST /api/applications` 创建申请 |
| PUT | 更新资源 | `PUT /api/applications/{id}` 更新申请 |
| DELETE | 删除资源 | `DELETE /api/applications/{id}` 撤销申请 |

### 1.2 统一响应格式

所有接口返回统一JSON格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "meta": {
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 业务状态码，200表示成功 |
| message | string | 提示信息 |
| data | object/array | 业务数据 |
| meta | object | 分页元数据（列表接口） |

### 1.3 认证方式

采用 **JWT (JSON Web Token)** 认证：

```
Authorization: Bearer <token>
```

**Token结构：**
```json
{
  "user_id": 1,
  "username": "zhangsan",
  "role": "applicant",
  "exp": 1712345678
}
```

**Token有效期：**
- Access Token: 2小时
- Refresh Token: 7天

### 1.4 错误码定义

| 错误码 | HTTP状态码 | 说明 |
|--------|-----------|------|
| 200 | 200 | 成功 |
| 400 | 400 | 参数错误 |
| 401 | 401 | 未登录或Token失效 |
| 403 | 403 | 无权限访问 |
| 404 | 404 | 资源不存在 |
| 409 | 409 | 资源冲突（如重复提交） |
| 422 | 422 | 业务校验失败 |
| 429 | 429 | 请求频率超限 |
| 500 | 500 | 服务器内部错误 |
| 503 | 503 | 服务暂不可用 |

## 3. 付款申请模块接口

### 3.1 创建付款申请

**接口路径：** `POST /api/applications`

**权限：** 部门申请人、会计

**请求参数：**
```json
{
  "payee_name": "北京XX科技有限公司",
  "payee_account": "6222021234567890123",
  "payee_bank": "中国工商银行北京分行",
  "amount": 50000.00,
  "purpose": "服务器采购款",
  "attachments": [
    {"file_id": "f_20260406_001", "file_name": "合同.pdf"},
    {"file_id": "f_20260406_002", "file_name": "发票.pdf"}
  ],
  "urgent": false,
  "remark": "请尽快处理"
}
```

**字段校验规则：**

| 字段 | 规则 |
|------|------|
| payee_name | 必填，长度≤200，不含特殊字符 |
| payee_account | 必填，银行卡号格式校验（Luhn算法） |
| payee_bank | 必填，长度≤200 |
| amount | 必填，范围0.01~999999999.99 |
| purpose | 必填，长度≤500 |
| attachments | 选填，单文件≤10MB |

**响应示例：**
```json
{
  "code": 200,
  "message": "申请提交成功",
  "data": {
    "id": 1,
    "application_no": "FK202604060001",
    "status": "pending",
    "created_at": "2026-04-06T10:30:00Z"
  }
}
```

### 3.2 查询申请列表

**接口路径：** `GET /api/applications`

**权限：** 所有角色（数据范围根据角色过滤）

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 否 | 状态筛选（draft/pending/approved/rejected/paid/verified） |
| department | string | 否 | 部门筛选 |
| start_date | date | 否 | 申请日期开始 |
| end_date | date | 否 | 申请日期结束 |
| keyword | string | 否 | 关键词搜索（户名、申请单号） |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页条数，默认20，最大100 |

**请求示例：**
```
GET /api/applications?status=pending&page=1&page_size=20
```

**响应示例：**

## 4. 会计审核模块接口

### 4.1 查询待审核列表

**接口路径：** `GET /api/applications/pending`

**权限：** 会计

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| department | string | 否 | 部门筛选 |
| amount_min | float | 否 | 金额下限 |
| amount_max | float | 否 | 金额上限 |
| urgent | bool | 否 | 是否紧急 |
| page | int | 否 | 页码 |
| page_size | int | 否 | 每页条数 |

**响应：** 同申请列表接口

### 4.2 审核通过

**接口路径：** `PUT /api/applications/{id}/approve`

**权限：** 会计（单笔<10万）、财务主管（单笔≥10万）

**请求参数：**
```json
{
  "note": "信息完整，审核通过"
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "审核通过",
  "data": {
    "id": 1,
    "application_no": "FK202604060001",
    "status": "approved",
    "approved_at": "2026-04-06T14:00:00Z",
    "approved_by": "李会计"
  }
}
```

### 4.3 审核拒绝

**接口路径：** `PUT /api/applications/{id}/reject`

**权限：** 会计、财务主管

**请求参数：**
```json
{
  "reason": "收款方账号格式错误，请核实后重新提交",
  "note": "账号末位可能有误"
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "审核已拒绝",
  "data": {
    "id": 1,
    "application_no": "FK202604060001",
    "status": "rejected",

  }
}
```

### 4.4 批量审核

**接口路径：** `POST /api/applications/batch-approve`

**权限：** 会计

**请求参数：**
```json
{
  "application_ids": [1, 2, 3, 4, 5],
  "action": "approve",
  "note": "批量审核通过"
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "批量审核完成",
  "data": {
    "success_count": 5,
    "failed_count": 0,
    "results": [
      {"id": 1, "status": "approved", "success": true},

    ]
  }
}
```

### 4.5 导出付款表

**接口路径：** `GET /api/applications/export`

**权限：** 会计

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| application_ids | array | 否 | 指定申请ID列表 |
| status | string | 否 | 按状态导出（approved） |
| start_date | date | 否 | 日期范围开始 |
| end_date | date | 否 | 日期范围结束 |
| format | string | 是 | 导出格式（excel/pdf/csv） |

**响应：** 返回文件流

```python
from rest_framework.response import Response

class ApiResponse:
    @staticmethod
    def success(data=None, message="success", meta=None):
        response = {
            "code": 200,
            "message": message,
            "data": data
        }
        if meta:
            response["meta"] = meta
        return Response(response)

    @staticmethod
    def error(code, message, data=None):
        return Response({
            "code": code,
            "message": message,
            "data": data
        }, status=code // 100)
```

### 10.4 对比引擎示例 (apps/comparison/engine.py)

```python
from difflib import SequenceMatcher

class ComparisonEngine:
    def compare(self, application, payment):
        result = {
            "is_match": True,
            "differences": []
        }

        # 1. 账号完全匹配
        if application.payee_account != payment.actual_payee_account:
            result["is_match"] = False
            result["differences"].append({
                "field": "账号",
                "expected": application.payee_account,
                "actual": payment.actual_payee_account,
                "severity": "CRITICAL"
            })

        # 2. 户名模糊匹配
        similarity = self._calculate_similarity(
            application.payee_name,
            payment.actual_payee_name
        )
        if similarity < 0.95:
            result["is_match"] = False
            result["differences"].append({
                "field": "户名",
                "expected": application.payee_name,
                "actual": payment.actual_payee_name,
                "similarity": similarity,
                "severity": "HIGH"
            })

        # 3. 金额精确匹配
        if abs(application.amount - payment.actual_amount) > 0.01:
            result["is_match"] = False
            result["differences"].append({
                "field": "金额",
                "expected": application.amount,
                "actual": payment.actual_amount,
                "difference": payment.actual_amount - application.amount,
                "severity": "CRITICAL"
            })

        return result

    def _calculate_similarity(self, str1, str2):
        # 去除空格和常见后缀
        def normalize(s):
            s = s.replace(" ", "")
            for suffix in ["有限公司", "有限责任公司", "股份有限公司"]:
                s = s.replace(suffix, "")
            return s
        return SequenceMatcher(None, normalize(str1), normalize(str2)).ratio()
```
